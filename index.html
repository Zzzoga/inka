ИНКА 4.0

ИНКА 4.0 — интегрированная модульная платформа, единая шина данных и инструментарий для разработки бизнес-приложений. Состоит из ядра и набора функциональных модулей, позволяющих создавать бизнес-продукты.

Основная функция ядра —  обеспечение коммуникации между  функциональными модулями. Состоит из 12 подсистем и портала для управления — это ключевые микросервисы, которые обеспечивают функционирование платформы, взаимодействие всех ее систем

Отвечает за регистрацию, управление и версионностью функциональных модулей и их расширениями в ядре.
BPMN движок
Подсистема управления бизнес логикой реализована на базе платформы Camunda с нашими доработками. Подсистема позволяет создавать и запускать бизнес процессы в рамках платформы, а также решать задачи по распределенным транзакциям.
UI движок
Позволяет создавать реактивные веб формы, на базе веб компонентов, с использованием данных о зарегистрированных в ядре сущностях и функциях.
Подсистема управления пользователями
Управление пользователями и ролями, а также управление правами на объекты доступа для ролей.
Подсистема уведомлений

Подсистема лицензирования

Подсистема мониторинга и логирования

Подсистема самотестирования
Осуществляет проверки подсистем ядра и функциональных модулей на их работоспособность
Портал ядра
Представляет собой единую точку входа для всех пользователей системы.
Подсистема печати

Часто задаваемые вопросы

Что такое платформа ИНКА 4.0 и для чего она предназначена?
ИНКА 4.0 — это цифровая платформа от НИЦ «КОНСОМ ГРУПП» для автоматизации и оптимизации производственных процессов, построения единой MES промышленных предприятий в любой отрасли.
Какие функциональные возможности предоставляет платформа ИНКА 4.0?
  • Микросервисную архитектуру для гибкости и масштабируемости.
  • Модульную структуру для настройки под потребности.
  • Ядро для коммуникации между модулями.
  • Инструменты для разработки бизнес-приложений.
  • Интеграционную шину для обработки данных.
Каковы преимущества использования платформы ИНКА 4.0 для разработки MES систем?
  • Гибкость и масштабируемость для быстрой адаптации к требованиям.
  • Интеграцию разных модулей для расширения системы.
  • Инструменты для создания уникальных модулей.
  • Интеграционную шину для связи с другими системами.
  • Применение в разных отраслях промышленности.
Какая архитектура у платформы ИНКА 4.0?
Платформа ИНКА 4.0 базируется на микросервисной архитектуре с Ядром и функциональными модулями. Взаимодействие с модулями осуществляется через DAPR. Функциональные модули представлены веб-сервисами, а доступ к данным обеспечивается протоколом oData.
Какие функциональные модули доступны на платформе ИНКА 4.0?

Платформа ИНКА 4.0 предлагает модули:

  • ИНКА.Интеграция: обеспечивает обмен данными в IT-инфраструктуре.
  • ИНКА.НСИ: стандартизация корпоративных справочников.
  • ИНКА.ЖД Логистика: учет и управление подвижным составом.
  • ИНКА. LIMS: управление качеством и лабораториями.
  • ИНКА.Заказы: управление заказами и производственной цепочкой.
  • ИНКА.Модель производства: учет производства и база MES системы с возможностью кастомизации.
